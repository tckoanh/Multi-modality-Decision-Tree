# Conclusion

In this tutorial, you explored how a decision tree works as part of **machine learning**, specifically within **supervised learning**. In supervised learning, the model is trained on data where the correct answers (labels) are already known, and it learns to map inputs (features) to outputs (labels).

You saw how a decision tree builds this mapping step by step by repeatedly splitting the data into smaller and cleaner groups. At each node, it chooses the best split using a measure like **information gain**, which reflects how much a split improves the separation of the classes. This process continues recursively until the data is well organized into final prediction groups.

You also developed an intuition for how this works across different types of data, from simple 2D visual cases to higher-dimensional and yes/no features, while keeping the same core idea: making the data easier to separate at every step.

Overall, decision trees provide a simple but powerful example of how supervised machine learning models learn patterns by asking a sequence of simple questions to gradually turn raw data into predictions.
